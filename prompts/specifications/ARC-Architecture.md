# ARC Architecture Specification

## 概要

**ARC Architecture** (Artificial Retro Code Architecture) は、仮想的なレトロハードウェアを構成するためのオープンかつ拡張可能な規格です。
特定の物理的な制約に縛られず、ソフトウェア上で自由な設計・組み変えが可能なモジュール群によって計算機システムを構築します。

## 設計思想とモジュラリティ

ARC Architecture の最大の特長は、その**高度なモジュラリティ（結合度の低さ）**にあります。

従来のエミュレータや仮想マシンが単一の巨大なプロセスとして各ハードウェアコンポーネントをエミュレーションするのに対し、ARC Architecture では以下の各コンポーネントが独立したプロセスまたはスレッド（モジュール）として振る舞います。

- **CPUモジュール**: 命令のフェッチと実行を担当。
- **VRAM/ディスプレイモジュール**: グラフィックスの描画と出力。
- **I/Oモジュール**: 外部デバイスとの入出力。
- **Memoryモジュール**: 主記憶へのアクセス管理。

これにより、ユーザーは用途に合わせて任意のモジュールを組み合わせ、自分だけの仮想ハードウェア（例えば、全く異なる命令セットを持つCPUや、独自のグラフィックス機能を持つVRAMなど）を容易に構築・拡張することが可能です。

## 通信バス (ZeroMQ)

ARC Architecture において、モジュール間の通信は**ZeroMQ**を中心としたメッセージパッシングによるシステムバスを通じて行われます。

1. **疎結合な連携**: ZeroMQのパブサブ(Pub/Sub)モデルやリクエスト/リプライモデルを利用することで、各モジュールはバスに対するメッセージの送受信のみを意識し、他のコンポーネントの内部実装に依存しません。
2. **多言語・分散対応**: ZeroMQのプロトコルを用いることで、異なるプログラミング言語で実装されたモジュールを組み合わせることや、ネットワーク越しに複数の物理マシンでモジュールを分散稼働させることも可能です。
3. **極めて強力なハックのしやすさ (Hackability)**: 通信バスが標準的なTCPやIPCソケットとして外部に開かれているため、サードパーティの自作モジュールや外部スクリプト（Pythonなど）を簡単に接続できます。これにより、内部メモリのリアルタイム監視、外部からの強制的なステート書き換え（チート作成）、機械学習エージェントの接続、カスタムデバッガの開発など、システム内部への干渉が容易に行える設計となっています。

## Neurom と Metov

### Neurom (ノイロム)
**Neurom**は、上記のARC Architecture規格に準拠して実装された**モジュール群の集合体・エミュレータ本体**の名称です。コアプロセス（今回は `neurom.exe` などのバイナリとして生成される）が各モジュールを管理・連携させ、統合された一つの仮想ハードウェアとして機能させます。

### Metov (メトフ)
**Metov**は、Neuromというプラットフォーム（仮想ハードウェア環境）の能力を示すために用意された**Neurom付属のサンプルプログラム（ゲーム）**の名称です。ユーザーは、Neurom上で動作するソフトウェアの一例としてMetovをプレイしながら内部パラメータをハックしたり、全く新しい自作のソフトを構築してNeurom上で動かすことができます。
